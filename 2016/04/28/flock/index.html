flock命令的一个坑

By Lirenke

Published Apr 28 2016

Contents

最近同事在工作中有用到linux的flock命令，我也是第一次听说，也不知道用法，网上一搜居然它也是常用命令，原来是孤陋寡闻了。

该命令主要给用户提供一个文件锁的功能，介质可以为一个文件或是一个目录，参数为：

Options:
 -s  --shared             get a shared lock
 -x  --exclusive          get an exclusive lock (default)
 -u  --unlock             remove a lock
 -n  --nonblock           fail rather than wait
 -w  --timeout <secs>     wait for a limited amount of time
 -E  --conflict-exit-code <number>  exit code after conflict or timeout
 -o  --close              close file descriptor before running command
 -c  --command <command>  run a single command string through the shell

 -h, --help display this help and exit
 -V, --version  output version information and exit

锁的类型可以是共享锁，或是一个独占锁，即排外锁；命令也可以设置为非阻塞的，默认阻塞，阻塞还可以设置一个超时时间。
题外话，补充一下共享锁和排外锁的概念，这里引用DB中的概念，同一个意思：

共享锁【S锁】
又称读锁，若事务T对数据对象A加上S锁，则事务T可以读A但不能修改A，其他事务只能再对A加S锁，而不能加X锁，直到T释放A上的S锁。这保证了其他事务可以读A，但在T释放A上的S锁之前不能对A做任何修改。

排他锁【X锁】
又称写锁。若事务T对数据对象A加上X锁，事务T可以读A也可以修改A，其他事务不能再对A加任何锁，直到T释放A上的锁。这保证了其他事务在T释放A上的锁之前不能再读取和修改A。

也就是说，如果先执行”flock -s”来锁住文件，其他进程就不允许再执行“flock -x”来独占，只可以再执行”flock -s”来获取，此为“共享”的含义；那么如果先执行”flock -x”来锁住文件，其他进程无论共享锁还是排它锁都无法获取到，会阻塞等待释放，或在非阻塞下直接返回失败。

关于用法，根据提示：

Usage:
 flock [options] <file|directory> <command> [<arguments>...]
 flock [options] <file|directory> -c <command>
 flock [options] <file descriptor number>

通过-c选项，可以让flock命令在获取文件锁后，立即执行一个命令。但在shell脚本中，无法指定为某个function，只能是可执行命令。

另外一个可以直接用“flock -s/-x ”来操作，意思是通过文件描述符（一个整数）来锁定对应的文件。Linux man page里有一段例子：

(
flock -s 200

# ... commands executed under lock ...

) 200>/var/lock/mylockfile

这里的>符号和200这个文件描述符的用法现在不是特别理解，大意便是打开”/var/lock/mylockfile”这个文件，文件描述符设置成200，通过指定fd的形式来做一个共享锁。

实际工作中，场景是有个shell脚本，其中想让某一段功能代码成为互斥区，即实现随时都只有一个进程能够执行那段代码。所以采用flock，但遇到了问题，当第一次在窗口A执行脚本后，成功拿到锁，同时在窗口B上执行脚本，B会等待互斥区代码结束，才会获取锁，往下走，这很正常。问题是，再同样在窗口A或B执行该脚本时，都发生了获取锁等待，即必然死锁，只有把原来使用的文件删除才能恢复。

很容易想到必然是其他进程占用了锁，lsof查看之发现问题所在了。原来那段互斥区代码中，启用了apache服务，httpd2进程作为shell脚本的子进程拷贝了父进程的所有文件描述符，包括已经打开的文件锁。当shell脚本结束后，apache服务自然成为了init进程的子进程，但锁也永远被其占用了。所以后面shell再怎么执行都将发生死锁。

如何解决呢？因为不知道shell下如何使子进程启动后关闭继承过来的文件描述符，只知道python的subprocess可以做到，用subprocess.Popen(cmd, close_fds=True)替换直接启动apache的bin文件解决问题。
