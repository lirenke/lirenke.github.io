如何迁移我的blog到gitcafe

By Lirenke

发表于 Jan 26 2015

文章目录

我的博客当前是使用github作为后台托管的，过程中发现当数据量渐渐变大时，每次调用“hexo g -d”发布时会很慢，原因是hexo在每次发布时都会重传全部的文件，这个很恶心！

另外还有个问题，就是当执行“hexo g -d”时，github会返回如下错误：
“fatal: the remote end hung up unexpectedly”。度娘以后，是可能文件太大导致，需要做如下更改：

解决方法：
windows:
在 .git/config 文件中加入
[http]
postBuffer = 524288000

linux:
git config http.postBuffer 524288000

我实际是在git bash下调用”git config –global –add http.postBuffer 524288000”搞定。

这个问题虽然得以解决，但是以后数据量大起来还得慢死。而且国内访问我的网站貌似比较卡。现在优化的想法有：

  1. 每次更新时，对生成的publish目录做diff，只上传变更的文件;
  2. 更换博客制作工具，比如用jekyll;
  3. 更换后台托管网站，采用国内访问更快的gitcafe;

第一个可以考虑；
第二个也可以考虑，只是现在hexo用习惯了，不想替换；
第三个是我刚实验的，很早就听说gitcafe了，国内的github；可以尝尝鲜。

gitcafe的操作和github基本一样，简直就是翻版，也可以支持个人pages，用法也一样，只不过从界面外观和文档的支持有点差距。所以同样的注册用户，上传公钥，创建page工程，在本地创建博客网站的源码，git push之即可。

至于dns服务器，我用的是dnspod，暂停原来指向github服务器的A条目，修改为指向gitcafe服务器，同时修改CNAME为gitcafe的域名，我的结果如下图所示：

最后，在本机进行试验时，记得清除浏览器的缓存。

当然github的后台我也不会完全弃用，gitcafe可以作为国内用户访问的推荐入口。（不过貌似现在没有用户 –！）
